Design System · Onboarding lineage

The system, unpacked.

A dark, utility-first language for Coinswap. Built around a single brand orange, a quiet neutral ramp, monospaced metadata, and components that earn their place. Every token below is in production on the Taker onboarding screen.

00 — Principles

What this system optimizes for.

Three rules every screen has to pass.
01

Utility over ornament.

No drop shadows for decoration, no gradients to hide a weak idea. Surfaces are flat, color is informational, motion happens only when state changes.

02

One signal color.

Orange is reserved. It marks primary action, active state, and forward progress — never as background filler. If two things are orange on a screen, one of them is wrong.

03

Mono for machine, sans for human.

JetBrains Mono carries technical metadata: ports, hashes, status codes, step counters. Inter carries everything a person reads as prose.

07 — Usage rules

Don't break the quiet.

A short list of things this system refuses to do.
Do
  • Use orange to lead the eye to one action per view.
  • Pair every technical value with a mono label.
  • Keep card surfaces flat — borders carry the hierarchy.
  • Let copy breathe — text-wrap:pretty + 1.55 line height.
Don't
  • Apply glass blur or gradient washes to surfaces.
  • Use orange for backgrounds, dividers, or icon fills.
  • Mix shadow depths inside a single card.
  • Use Inter for ports, hashes, or step counters.
